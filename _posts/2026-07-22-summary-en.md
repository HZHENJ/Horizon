## [SkewAdam: Tiered Optimizer Cuts MoE State Memory by 97%](https://www.reddit.com/r/MachineLearning/comments/1v38k1m/skewadam_a_tiered_optimizer_that_cuts_moe_state/) ⭐️ 8.0/10

SkewAdam, a new optimizer for Mixture-of-Experts (MoE) training, employs a tiered state allocation strategy: backbone parameters keep momentum and factored second moment, experts use only factored second moment, and the router gets exact second moment. This reduces optimizer state memory from 50.6 GB to 1.29 GB (a 97.4% reduction) and peak training memory from 81.4 GB to 31.3 GB, enabling a 6.78B MoE model to fit on a single 40GB GPU. This advance breaks a major memory bottleneck in MoE training, where optimizer states often dominate memory usage. By enabling consumer-grade GPUs to train large MoE models, it democratizes access to powerful architectures previously limited to clusters. The tiered allocation is motivated by parameter behavior: backbone parameters are critical for representational power and thus get richer states, while experts, being numerous but individually less sensitive, get lean states. The router, with very few parameters, uses exact second moment to maintain convergence stability. The reduction is orthogonal to parameter quantization and does not involve compressing states to 8-bit, avoiding the int32 boundary issue that plagues 8-bit optimizers.

reddit · r/MachineLearning · /u/Kooky-Ad-4124 · Jul 22, 07:04

**Impact**: Short-term, researchers and practitioners can train 6.7B-parameter MoEs on a single 40GB GPU like the NVIDIA A100-40GB, significantly lowering hardware barriers and energy costs. Longer-term, this technique could be integrated into deep learning frameworks, encouraging wider adoption of MoE architectures in resource-constrained settings and potentially influencing the design of future memory-efficient optimizers for even larger models.

**Background**: Mixture-of-Experts (MoE) models divide a neural network into multiple sub-networks (experts) that specialize in different inputs, activating only a subset per forward pass, which allows scaling parameter count without proportionally increasing compute. Optimizers like Adam keep moving averages of gradients (first and second moments) per parameter, which effectively doubles or triples the memory compared to the model weights. For large models, this optimizer state dominates VRAM usage. Adafactor is a memory-efficient optimizer that uses factored second moment estimates, reducing memory by storing vectors instead of full matrices. SkewAdam builds on these ideas with a tiered approach.

## [Sandbox Escapes Found in Four Major AI Coding Agents via Indirect Prompt Injection](https://www.bleepingcomputer.com/news/security/cursor-codex-gemini-cli-antigravity-hit-by-sandbox-escapes/) ⭐️ 8.0/10

Security researchers at Pillar Security have disclosed sandbox escape vulnerabilities in four major AI coding agents: Cursor, OpenAI Codex, Google Gemini CLI, and Antigravity. Attackers can exploit these flaws by embedding malicious prompts in repository files (e.g., README, issues) via indirect prompt injection, causing the agents to generate files that execute arbitrary code on developers' machines. This discovery is significant because it demonstrates a novel attack vector—'sandbox escape without escape'—that exploits the trust between AI coding agents and local development tools, bypassing isolation mechanisms that were thought to be secure. As AI agents become deeply integrated into software development workflows, such vulnerabilities could lead to widespread supply chain attacks and developer machine compromises. The attack relies on indirect prompt injection: malicious prompts embedded in repository content cause the AI agent to write files that are automatically executed by host tools like Python interpreters or Git hooks. Cursor fixed in version 3.0.0 and Codex CLI in v0.95.0, while Google deems Antigravity's two vulnerabilities as lower severity requiring social engineering.

telegram · zaihuapd · Jul 22, 08:08

**Impact**: In the short term, developers using vulnerable versions of these AI coding agents risk arbitrary code execution when working with malicious repositories. The immediate patches from Cursor and OpenAI mitigate the risk, but Google's downgrade of Antigravity's flaws suggests some vendors may not fully address the threat. Long-term, this will likely drive the adoption of more robust sandboxing techniques and increased scrutiny of how AI-generated files are consumed by local toolchains.

**Background**: AI coding agents are tools that use large language models to assist in software development, often running in sandboxes to isolate their operations from the host system. Indirect prompt injection is an attack where malicious instructions are hidden in data that the AI processes, such as website content or repository files, causing it to perform unintended actions. Traditional sandboxes focus on preventing direct access to the host, but these attacks exploit the fact that files generated by the agent inside the sandbox are later executed by trusted host tools.
